# accounts update

Command: `boundary accounts update`

The `accounts update` command updates an existing account's information for
LDAP, OIDC, or password auth method.

## Examples

An existing password-type account (`acctpw_w3N3PlV34D`) has the following
detail. 

```plaintext
ID:                    acctpw_w3N3PlV34D
   Version:             1
   Type:                password
   Name:                test_account
   Description:         Test password account
   Authorized Actions:
      no-op
      read
      update
      delete
      set-password
      change-password

  Attributes:
    Login Name:        tester01
```

Update the name, login name, and description.

```shell-session
$ boundary accounts update password -id=acctpw_w3N3PlV34D \
   -name="tester" \
   -login-name="test-engineer" \
   -description="Only to be used for testing"
```

## Usage

<CodeBlockConfig hideClipboard>

```shell-session
$ boundary accounts update [type] [options] [args]
```

</CodeBlockConfig>

The available types are: `ldap`, `oidc`, and `password`.

### Command options

- `-description` `(string: "")` - Description to set on the account.
- `-id` `(string: "")` - ID of the account on which to operate.
- `-name` `(string: "")` - Name to set on the account.
- `-version` `(string: "")` - The version of the account against which to
   perform an update operation. If not specified, the command will perform a
   check-and-set automatically.

options.

- `-login-name` `(string: "")` - The login name for the account
